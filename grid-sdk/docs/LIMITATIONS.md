# GridSdk 제한사항 및 Known Issues

## PRD 기반 제한

### 최적의 공간 없을 시 이동/리사이즈 미지원

- 이동 또는 리사이즈 시 충돌하는 아이템을 재배치할 **유효한 후보**가 없으면 해당 동작은 **취소**됩니다.
- 기존 상태가 유지되며, `EngineResult.Failure`(예: `NoFeasibleLayout`)가 반환됩니다.
- 예: 4×4 그리드에 2×2 아이템 4개가 꽉 차 있을 때, 한 아이템을 중앙으로 이동 시도 → 실패

### 그리드 축소 시 경계 밖 아이템

- 그리드 행/열을 런타임에 축소할 때, 경계를 벗어나는 아이템에 대한 **공식 재배치**는 제공하지 않습니다.
- 앱에서 `GridSize` 변경 시 `items.filter { it.isValidIn(newGridSize) }` 등으로 **경계 밖 아이템을 제거**하는 정책을 적용해야 합니다.

### 롤백 규칙

- **드래그 중**에만 롤백이 적용됩니다. 재배치된 아이템이 원위치로 복귀 가능하면 `GridLayout` 내부에서 `evaluateRollback`이 호출되어 자동 복귀합니다.
- **프로그래매틱** Move/Resize(`GridEngine.process` 직접 호출) 후에는 별도 롤백 API가 없습니다.

## 성능

### 대량 아이템

- 아이템 수가 많을 때(예: 50개 이상) 드래그/리사이즈 중 **매 프레임** 배치 계산이 수행되므로, 저사양 기기에서 **프레임 드랍**이 발생할 수 있습니다.
- 배치 계산은 결정적(deterministic)이며 최적화되어 있으나, 극단적으로 많은 아이템에서는 주의가 필요합니다.
