# 프로젝트 주제
   Jetpack Compose를 이용해서 아이템을 그리드 형태로 배치할 수 있는 레이아웃을 구현해보자.
   아이템은 각자 x,y좌표 그리고 x,y 방향의 Span 값을 가진다. 
   최종 목표는 해당 기능을 위해 sdk로 만들어서 사용하고자하는 앱에 예제와 함께 가이드 한다

# 그리드 모델
- 그리드는 고정 `N x M` 구조를 사용한다.
- `N`(행), `M`(열)은 설정값으로 변경 가능하다.

# 기능
- 아이템 추가 및 삭제
- Drag로 아이템을 이동
- Long Press -> Drag로 아이템 사이즈 변경(Span이 업데이트될때마다) 할수 있는 리사이즈 기능

# 특화 기능
- 아이템을 추가하면 남은 공간에 알아서 배치된다.
- Drag로 아이템을 이동시키는데 Drop하는 공간에 다른 아이템이 있다면 다른 아이템들이 Drop 공간을 제외한 영역에 재배치된다.
- 아이템을 리사이즈함으로써 이미 위치해있던 아이템의 공간을 침범한다면 침범당한 아이템들은 다른 최적의 공간에 재배치된다.
- 재배치되는 모든 전제는 최적의 공간을 찾을수 있을때를 가정한다. 최적의 공간이 없을경우는 재배치가 안되고 이동 및 리사이즈도 지원하지 않는다.
- 최적의 공간 판단 우선순위는 아래와 같다.
  1. 재배치되는 아이템 수 최소
  2. 드래그/리사이즈 목표 위치와의 맨해튼 거리 최소
  3. 동률이면 상단-좌측 우선

# 추가 기능
- 아이템이 이동 및 리사이즈됨으로써 기존의 위치 데이터가 아닌 다른 위치로 변경된 아이템들에 한해서 드래그 중인 아이템이
  다시 다른 공간으로 감으로써 기존 위치로 다시 돌아갈수 있으면 기존 위치로 롤백한다.(기존 사용성을 위해)
- 아이템들의 이동은 seamless하게 애니메이션이 지원된다

# 알고리즘 스펙
## 데이터 모델
- Grid: `N x M` 고정 셀
- Item: `id`, `x`, `y`, `spanX`, `spanY`
- Occupancy: 각 셀의 점유 아이템 `id` 또는 빈 값

## 공통 제약
- 모든 아이템은 그리드 경계를 벗어나면 안된다.
- 아이템 간 셀 중복 점유는 허용하지 않는다.
- 후보 배치가 위 제약을 만족할 때만 유효 후보로 본다.

## 입력/출력
- 입력: 현재 아이템 목록, 드래그/리사이즈 대상 아이템, 목표 좌표(또는 목표 Span), `N`, `M`
- 출력:
  - 성공: 대상 아이템 최종 위치/크기 + 재배치된 아이템들의 최종 위치
  - 실패: 기존 상태 유지(이동/리사이즈 미적용)

## 후보 탐색 규칙
- 탐색 대상은 대상 아이템의 목표 상태를 기준으로 충돌을 해소할 수 있는 전체 배치 상태다.
- 유효 후보 여러 개가 있으면 아래 우선순위로 1개를 선택한다.
  1. 재배치되는 아이템 수 최소
  2. 목표 위치와의 맨해튼 거리 최소
  3. 동률이면 상단-좌측 우선(행 우선, 열 우선)

## 동작별 처리
- 아이템 추가:
  - 상단-좌측부터 빈 영역을 탐색해 아이템이 들어갈 수 있는 첫 위치에 배치한다.
  - 가능한 위치가 없으면 추가를 거부한다.
- 아이템 이동(Drag):
  - 대상 아이템을 목표 위치에 두고 충돌 아이템 재배치 후보를 계산한다.
  - 유효 후보가 있으면 우선순위에 따라 적용한다.
  - 유효 후보가 없으면 이동을 취소한다.
- 아이템 리사이즈(Long Press -> Drag):
  - 목표 `spanX`, `spanY`로 확장/축소한 뒤 충돌 아이템 재배치 후보를 계산한다.
  - 유효 후보가 있으면 우선순위에 따라 적용한다.
  - 유효 후보가 없으면 리사이즈를 취소한다.

## 롤백 규칙(드래그 중)
- 드래그 중 재배치되었던 아이템은, 이후 포인터 이동으로 인해 본래 위치가 다시 유효해지면 원위치로 복귀시킨다.
- 복귀 판단은 현재 프레임의 점유 상태 기준으로 수행한다.

## 실패 조건
- 경계 초과, 중복 점유, 또는 재배치 가능한 유효 후보 없음
- 실패 시 항상 기존 상태를 보존한다.

## 성능/렌더링 기준
- 드래그/리사이즈 중 매 프레임 재계산이 가능하도록 배치 계산은 결정적(deterministic)이고 빠르게 동작해야 한다.
- 위치 변화는 아이템 단위 애니메이션으로 연결해 시각적으로 seamless하게 보이도록 한다.

# 유사 사례
